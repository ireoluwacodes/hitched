# Hitched — QR Table Ordering

QR-code-based food ordering for events and weddings. Guests scan a table QR code, pick menu items on their phone, and orders appear instantly on the staff dashboard.

## Stack

- Vite + React 19 + TypeScript
- Convex (real-time backend)
- shadcn/ui (radix-mira, mauve theme)

## Getting started

### 1. Install dependencies

```bash
bun install
```

### 2. Start Convex

In one terminal:

```bash
bunx convex dev
```

This creates `.env.local` with `VITE_CONVEX_URL` if it doesn't exist.

On first load, the app auto-seeds sample data (dev PIN: `1234`, 10 tables, Nigerian wedding menu).

To manually re-seed:

```bash
bunx convex run seed:default
```

### 3. Start the frontend

In another terminal:

```bash
bun run dev
```

Open [http://localhost:5173](http://localhost:5173).

## Routes

| Route | Description |
|---|---|
| `/` | Landing page |
| `/t/:qrToken` | Guest order page (scan QR) |
| `/t/:qrToken/order/:orderId` | Edit a pending order |
| `/admin` | Staff PIN gate |
| `/admin/dashboard` | Live kitchen order display |
| `/admin/menu` | Menu management |
| `/admin/tables` | Table count + QR PDF export |
| `/admin/settings` | Event name, ordering toggle, PIN |

## Environment

```bash
# .env.local (auto-generated by convex dev)
VITE_CONVEX_URL=https://your-deployment.convex.cloud

# Optional: production URL for QR codes in PDF export / table preview
# For local phone testing, use Vite's Network URL (same Wi‑Fi required):
VITE_APP_URL=http://192.168.1.124:5173
```

## Default dev credentials

- Admin PIN: `1234`
- Tables: 1–10 (each with unique QR token)
- View QR codes at `/admin/tables` after logging in
